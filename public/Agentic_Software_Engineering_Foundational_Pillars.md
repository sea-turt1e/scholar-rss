## 論文情報

- **著者**: AE Hassan, H Li, D Lin, B Adams, TH Chen
- **論文概要リンク**: https://arxiv.org/abs/2509.06216
- **論文PDFリンク**: https://arxiv.org/pdf/2509.06216

## 要約

Agentic Software Engineering（以下、SE 3.0）は、単なるコード生成を超え、複雑かつ目標志向的なソフトウェアエンジニアリングをAIエージェントと人間が共に達成する新時代のパラダイムを指す。本論文では、この新時代を実現するため、SEの基盤となる役者（Actors）、プロセス（Processes）、ツール（Tools）、アーティファクト（Artifacts）を根本的に再構築する必要性を説く。具体的には、人間とエージェント双方の異なるモダリティを同時に扱う「Structured Agentic Software Engineering（SASE）」フレームワークを提案し、これを支える専用ツール環境や版管理された構造化アーティファクト群を示す。また、この分野が突き当たる課題と研究ロードマップを提示し、教育面への影響にも言及している。

## 主要なポイント

1. SE 3.0は人間中心のソフト開発プロセスを超え、多数の自律的AIエージェントと人間が協働するN対Nのソフトウェア開発環境を求めている。
2. 既存のIDEや単発のプロンプトベースAI支援は限界があり、専用の「Agent Command Environment（ACE）」や「Agent Execution Environment（AEE）」のような二種のワークベンチが不可欠である。
3. エンジニアリング活動は「BriefingEng（明確な仕様書作成）」「Agentic Loop Engineering（ループ指揮）」「MentorScript（エージェント指導コード化）」「Agentic Guidance Engineering（人間-エージェント間の対話最適化）」など複数の構造化活動に分割される。
4. AIエージェントのエージェント自身のライフサイクル管理や専用インフラ整備も重要であり、エージェント・ファーストのコード管理や複雑なマルチエージェントの協調が今後の鍵となる。
5. この新パラダイムは、人間エンジニアの役割を単なるコード実装者から「エージェントコーチ」へと大きく変革し、ソフトウェア工学教育の全面的な再設計を促す。


## メソッド

- **構造化二元論（Duality）**：SEは「SE for Humans（人間中心）」と「SE for Agents（エージェント中心）」の二軸で再定義される。これにより、役者、プロセス、アーティファクト、ツールがそれぞれのモダリティで異なる形態・役割を担う。
- **アーティファクト設計**：仕様書である「BriefingScript」、実行手順を定義する「LoopScript」、チーム指導原則を規定する「MentorScript」、エージェント発信の相談用「Consultation Request Pack（CRP）」、マージ準備完了証明の「Merge-Readiness Pack（MRP）」、そしてそれらに対する人間の応答を管理する「Version Controlled Resolutions（VCR）」などを開発。これらは全て構造化され版管理される。
- **専用ワークベンチ**：
- 「Agent Command Environment (ACE)」は人間コーチが多エージェントを監督、指導するための環境であり、構造化された情報提示や複数ソリューションの比較、エージェントチーム編成も行う。
- 「Agent Execution Environment (AEE)」はエージェントがタスクを実行・評価・監視する場。大規模並列計算やセマンティック検索、高度なデバッグツールを備え、エージェントに最適化されている。
- **連続的対話ループ**：人間とAIはBriefingScriptでタスクを開始し、LoopScriptに従ってエージェントが並列作業。CRPを通じてエージェントは人間の専門知識を呼び出し、MRPで証明付き成果物を返す。人間はVCRでこれに応答し、全行程を生きたドキュメントとして管理。

## 意義・影響

- AIエージェントによる単なるコード生成ではなく、信頼性・保守性の高いソフトウェアを人間とAIが共同で創り上げるための基盤設計を初めて体系化した。
- 人間中心のソフトウェア開発から、人間と多様なAIエージェントが役割を分担し協調する新たな工学パラダイムへと転換を促す。
- 「マージ準備証明」や「指導コード」などの新しい構造化アーティファクトが信頼性担保や監査性を高め、ソフトウェア品質向上に寄与する。
- ソフトウェアエンジニアの役割変革（コード書き手からエージェントコーチへ）と、それに伴う教育プログラムの刷新が必須となることを強調し、未来のSE人材育成に大きな示唆を与える。
- 多数エージェントと大規模協調するためのインフラ技術や継続的学習・メモリ管理など新領域の研究課題を明らかにし、研究コミュニティの方向性を提示した。
